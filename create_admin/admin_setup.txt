ADMIN ACCOUNT SETUP GUIDE
=========================

This guide explains the current admin creation flow for the E-Class Record System.

CURRENT SCRIPT
==============

Use this single script:

create_admin/create_admin.py

The old duplicate script was removed. Do not use create_admin_pymysql.py.

WHAT THE SCRIPT DOES
====================

When creating an admin, the script now ensures:

1. A users row exists with role=admin.
2. A personal_info row exists with an email.
3. An instructors row is linked to that personal_info record.

This linkage is required so admin MFA email delivery can work.

DEFAULT VALUES FROM .env
========================

Required:

- DEFAULT_ADMIN_USERNAME
- DEFAULT_ADMIN_PASSWORD

Optional (recommended):

- DEFAULT_ADMIN_EMAIL (default: programmingproject06@gmail.com)
- DEFAULT_ADMIN_FIRST_NAME (default: System)
- DEFAULT_ADMIN_LAST_NAME (default: Administrator)
- DEFAULT_ADMIN_DEPARTMENT (default: Administration)

If DEFAULT_ADMIN_EMAIL is missing, the script uses programmingproject06@gmail.com.

MFA + EMAIL REQUIREMENTS
========================

If MFA is enabled for admin accounts, make sure these are configured in .env:

- MFA_ENABLED=true
- MFA_REQUIRED_ROLES includes admin
- SMTP_SERVER
- SMTP_PORT
- SENDER_EMAIL
- SENDER_PASSWORD

Without working SMTP credentials, MFA code delivery will fail.

CREATE THE DEFAULT ADMIN
========================

1. Activate your virtual environment.
2. Run:

    python create_admin/create_admin.py

3. Click Create Default Admin.

The script will create or update the admin linkage and show the resolved email used for MFA.

CREATE A CUSTOM ADMIN
=====================

1. Run the same script.
2. Click Create Custom Admin.
3. Enter School ID, Password, Confirm Password, and Email.

The custom flow also creates the required personal_info/instructors linkage.

LOGIN URL
=========

Use this path for admin login:

http://127.0.0.1:5000/admin-login

TROUBLESHOOTING
===============

If admin creation fails:

1. Check database connection values in .env.
2. Ensure DEFAULT_ADMIN_PASSWORD is set.
3. If email is already used by another personal_info record, choose a different admin email.
4. Check app logs for SQL/constraint errors.

If admin login fails with MFA:

1. Confirm admin email linkage exists.
2. Confirm SMTP credentials are valid.
3. Verify MFA env flags are set and app was restarted.
